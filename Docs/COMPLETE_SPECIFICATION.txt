PART II: COMPLEXITY MEASURES
================================================================================

1. IDEAL KOLMOGOROV COMPLEXITY
-------------------------------

For any entity e, we define:

K(e) = Ideal Kolmogorov complexity of e
     = Length of shortest description of e in bits
     = Algorithmic information content

Properties:
  K(e) ≥ 0 for all e
  K(Substrate) = 0 (substrate is maximally simple)
  K(e) > 0 for all presentations e

JOINT COMPLEXITY
  K_joint(e₁, e₂, ..., eₙ) = Complexity of the collection as a whole
  
CONDITIONAL COMPLEXITY  
  K(e₁ | e₂) = K_joint(e₁, e₂) - K(e₂)
             = Additional information needed to specify e₁ given e₂

SUM OF COMPLEXITIES
  K_sum(e₁, e₂, ..., eₙ) = K(e₁) + K(e₂) + ... + K(eₙ)

2. OPERATIONAL LEMPEL-ZIV COMPLEXITY
-------------------------------------

For binary strings (states), we use an operational measure:

K_LZ(s) = Lempel-Ziv complexity of state s
        = Number of distinct patterns in s
        = Computable approximation to ideal complexity

Properties:
  K_LZ(empty string) = 0
  K_LZ increases with string length
  K_LZ approximates K in the limit

3. TOPOLOGICAL COMPLEXITY
--------------------------

For manifolds, we define:

K_topo(M) = Algorithmic complexity of manifold M's topological structure
          = Information needed to specify M's geometry

For the substrate manifold:

K_topo(M_substrate) = K_base + K_orbifold + K_resolution + K_geometry

Where:
  K_base       = 3 bits   (six-torus T⁶ structure)
  K_orbifold   = 4 bits   (ℤ₃ × ℤ₃ quotient action)
  K_resolution = 10 bits  (27 exceptional divisor blowups)
  K_geometry   = 40 bits  (Hodge numbers, metric, moduli)
  ─────────────────────────
  Total        = 57 bits

4. COHERENCE MEASURE
--------------------

For a collection of entities over time, we define:

Coherence(entities, times) = 1 - K_joint(entities@times) / K_sum(entities@times)

This measures how much the entities are "locked together" versus independent.

Properties:
  0 ≤ Coherence ≤ 1
  Coherence = 1 means perfect correlation (maximal quantum behavior)
  Coherence = 0 means independence (classical behavior)

PART X: MATHEMATICAL FOUNDATIONS
================================================================================

1. KOLMOGOROV COMPLEXITY BASICS
--------------------------------

DEFINITION:
  K(x) = Length of shortest program that outputs x
       = min{|p| : U(p) = x}
  where U is a universal Turing machine.

PROPERTIES:
  - Non-computable (halting problem)
  - Invariant up to additive constant (choice of U)
  - K(x) ≤ |x| + O(1) (can always output literally)
  - K(xy) ≤ K(x) + K(y) + 2log(min(K(x),K(y))) + O(1)

PHYSICAL RELEVANCE:
  K measures "true randomness" or "incompressibility"
  Objects with low K have structure/patterns
  Objects with high K appear random

2. CONDITIONAL COMPLEXITY
--------------------------

DEFINITION:
  K(x|y) = Length of shortest program that outputs x given y as input
         = K(x,y) - K(y) + O(1)

PROPERTIES:
  - K(x|y) ≤ K(x) + O(1)
  - K(x|x) = O(1) (given x, outputting x is trivial)
  - K(x,y) ≤ K(x) + K(y|x) + O(1)

INFORMATION:
  K(x|y) measures information in x not present in y
  If K(x|y) ≈ 0, then y contains all information about x

3. MUTUAL INFORMATION
---------------------

DEFINITION:
  I(x:y) = K(x) + K(y) - K(x,y)
         = Amount of shared information

PROPERTIES:
  - I(x:y) ≥ 0 (non-negative)
  - I(x:y) = I(y:x) (symmetric)
  - I(x:y) ≤ min(K(x), K(y))

SUBSTRATE THEORY:
  Inseparability ⟺ I(e₁:e₂) > 0
  Strong inseparability ⟺ I(e₁:e₂) ≈ min(K(e₁), K(e₂))

4. LEMPEL-ZIV COMPRESSION
--------------------------

ALGORITHM:
  Parse string left-to-right into non-overlapping blocks.
  Each block is shortest substring not seen before.
  K_LZ = number of blocks

EXAMPLE:
  s = "0001010011010"
  Parsing: 0|00|1|01|001|10|10
  K_LZ(s) = 7

CONVERGENCE:
  K_LZ(s) / |s| → h(source) as |s| → ∞
  where h(source) is entropy rate

PRACTICAL USE:
  Computable approximation to ideal Kolmogorov complexity
